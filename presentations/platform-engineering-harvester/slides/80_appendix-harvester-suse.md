# <span class="accent">Harvester</span> — HCI by SUSE<span class="dot">.</span>

<p class="lede" style="margin-top: 20px; max-width: 80ch;">
Open-source Hyperconverged Infrastructure from SUSE — run virtual machines and containers on bare metal, managed the Kubernetes way.
</p>

<div style="display: grid; grid-template-columns: repeat(2, 1fr); gap: 24px; margin-top: 40px;">
  <div class="surface" style="padding: 28px;">
    <div style="font-size: 20px; font-weight: 600;">Kubernetes-native</div>
    <p style="font-size: 17px; line-height: 1.45; margin-top: 8px; color: var(--fg-muted);">
      Built on KubeVirt (VMs) and Longhorn (storage). Everything is a Kubernetes resource — driven by UI or API.
    </p>
  </div>
  <div class="surface" style="padding: 28px;">
    <div style="font-size: 20px; font-weight: 600;">Bare-metal HCI</div>
    <p style="font-size: 17px; line-height: 1.45; margin-top: 8px; color: var(--fg-muted);">
      Compute, storage and networking in one install — no external SAN. VMs and containers side by side.
    </p>
  </div>
  <div class="surface" style="padding: 28px;">
    <div style="font-size: 20px; font-weight: 600;">Rancher-integrated</div>
    <p style="font-size: 17px; line-height: 1.45; margin-top: 8px; color: var(--fg-muted);">
      Managed as a node driver in Rancher — provision guest Kubernetes clusters straight onto Harvester.
    </p>
  </div>
  <div class="surface" style="padding: 28px;">
    <div style="font-size: 20px; font-weight: 600;">Open source</div>
    <p style="font-size: 17px; line-height: 1.45; margin-top: 8px; color: var(--fg-muted);">
      Apache-2.0, vendor-neutral, backed by SUSE — no per-socket licensing.
    </p>
  </div>
</div>
